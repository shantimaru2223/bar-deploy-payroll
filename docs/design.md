# 設計書 - Bar Deploy 給与明細アプリ

## 1. 技術スタック

| 項目 | 技術 | 選定理由 |
|---|---|---|
| フロントエンド | HTML / CSS / JavaScript | シンプルで学習コストが低い |
| バックエンド | Node.js（Express） | 軽量で小規模アプリに最適 |
| データベース | SQLite | サーバー不要、ファイル1つで管理できる |
| PDF出力 | pdf-lib または pdfmake | JavaScript だけでPDF生成が可能 |

> **比喩**: 技術スタックとは「お店を建てるときの建材の選択」のようなものです。
> 木造（シンプル）か鉄骨（高機能）か、お店の規模に合った材料を選びます。

## 2. 画面構成

```
[トップページ]
  ├── スタッフ一覧・管理画面
  │     ├── スタッフ登録フォーム
  │     └── スタッフ編集・削除
  ├── 勤怠入力画面
  │     ├── 対象月の選択
  │     └── 日ごとの勤務時間入力
  └── 給与明細画面
        ├── 明細プレビュー
        └── PDF出力ボタン
```

## 3. データ設計

### 3.1 スタッフテーブル（staff）
| カラム名 | 型 | 説明 |
|---|---|---|
| id | INTEGER | 主キー（自動採番） |
| name | TEXT | スタッフ名 |
| pay_type | TEXT | 給与タイプ（"hourly" または "monthly"） |
| hourly_rate | INTEGER | 時給（円）※ 時給スタッフのみ使用 |
| monthly_salary | INTEGER | 月給額（円）※ 月給スタッフのみ使用 |
| transport_fee | INTEGER | 月額交通費（円） |
| created_at | TEXT | 登録日時 |

### 3.2 勤怠テーブル（attendance）
| カラム名 | 型 | 説明 |
|---|---|---|
| id | INTEGER | 主キー（自動採番） |
| staff_id | INTEGER | スタッフID（外部キー） |
| work_date | TEXT | 勤務日（YYYY-MM-DD） |
| start_time | TEXT | 開始時間（HH:MM） |
| end_time | TEXT | 終了時間（HH:MM） |
| work_hours | INTEGER | 勤務時間（分単位で保存） |

> **比喩**: テーブルとは「Excelのシート」のようなものです。
> スタッフの情報シートと、出勤記録シートの2つを用意するイメージです。

## 4. 給与計算ロジック

```
1. 対象月の勤怠データを取得
2. スタッフの給与タイプを確認

【時給スタッフの場合】
3a. 各日の勤務時間（分）を合計 → 総勤務分数
4a. 基本給 = Math.floor(総勤務分数 / 60 * 時給)
    ※ 1分単位で正確に計算し、端数は切り捨て

【月給スタッフの場合】
3b. 基本給 = 月給額（そのまま）

5. 総支給額 = 基本給 + 交通費
6. 源泉徴収税額 = Math.floor(基本給 * 0.1021)
   ※ 基本給の10.21%、端数切り捨て
7. 差引支給額 = 総支給額 - 源泉徴収税額
```

## 5. PDF出力の内容

### 時給スタッフの場合
```

        給 与 明 細 書
====================================
  店舗名:   Bar Deploy
  対象期間: 2026年2月分
------------------------------------
  スタッフ名: 山田太郎
  給与タイプ: 時給
  時給:       1,200円
------------------------------------
  勤務日数:     15日
  総勤務時間:   90時間30分
  基本給:     108,600円
  交通費:       5,000円
  総支給額:   113,600円
------------------------------------
  【控除】
  源泉徴収税額（10.21%）: -11,088円
------------------------------------
  差引支給額: 102,512円
====================================
```
